# 经历成就产品 PRD 草稿

## 1. 产品定位

这是一个把用户真实生活经历转化为个人奖章、人生档案和社交展示内容的 AI 产品。

产品核心不是传统日记，也不是纯 AI 陪伴，而是帮助用户从真实经历中获得成就感、意义感和被理解感。

核心价值：

- 用户上传或讲述自己的经历。
- Agent 理解经历中的行动、情绪、意义和人格特质。
- 系统生成一枚贴合经历的主奖章。
- 奖章沉淀到个人主页、时间线和成就档案中。
- 用户可以选择公开展示，与他人产生轻互动。

一句话描述：

> 每个人都可以把自己的真实经历转化成值得纪念和展示的个人成就奖章。

## 2. 目标用户

第一版不按垂直人群切分，面向所有希望从真实经历中获得成就感的人。

典型使用动机：

- 希望自己的生活被确认和看见。
- 希望把普通经历变得更有意义。
- 希望积累一个长期的人生经历档案。
- 希望通过奖章展示自己的生活、成长和个性。

## 3. MVP 范围

### 3.1 MVP 核心闭环

1. 用户通过文字、语音或图片讲述一段经历。
2. Agent 通过对话追问，理解经历。
3. Agent 先给出理解总结，用户确认。
4. Agent 生成一枚主奖章。
5. 用户查看奖章外层信息和详情叙事。
6. 用户可调整视觉、编辑内容、设置公开范围。
7. 奖章进入个人主页、时间线和社交流。

### 3.2 MVP 输入方式

MVP 先以对话式输入为主，同时支持：

- 文字输入
- 语音输入，转文字后进入 Agent 对话
- 图片上传

视频上传、完整语音通话、Agent 语音回复放到后续迭代。

### 3.3 MVP 核心页面

1. 首页 / 社交流
2. 创建经历
3. 奖章详情
4. 个人主页
5. 消息 / 通知
6. Agent 设置

## 4. 核心用户流程

### 4.1 创建经历

用户进入创建页后，可以通过文字、语音或图片开始记录经历。

Agent 默认采用“专业采访 + 轻陪伴”的风格：

- 不过度鸡汤
- 不过度游戏化
- 重点帮助用户把经历讲清楚
- 适度表达理解和陪伴

Agent 追问数量根据经历复杂度动态决定。用户始终可以跳过追问，直接生成奖章。

### 4.2 生成前确认

在生成奖章前，Agent 先给出对经历的理解总结。

用户可以：

- 确认并生成奖章
- 继续补充信息
- 修改 Agent 的理解
- 直接点击“生成奖章”

### 4.3 生成奖章

一次经历默认生成一枚主奖章。

Agent 决定这段经历最值得被授予什么奖章，也就是奖章的意义重心。

用户可以调整视觉样式，但不改变奖章主题含义本身，除非用户主动进行“意义重生成”。

## 5. 奖章系统

### 5.1 奖章生成原则

奖章不固定奖励某一种内容，而是由 Agent 根据经历自动判断。

可能的奖章重心包括：

- 行动成就：用户完成了什么
- 情绪成就：用户经历了什么情绪并走过来
- 人格特质：这件事体现了怎样的品质
- 关系意义：这件事和他人的连接
- 成长变化：这件事带来的突破或转变

### 5.2 Agent 内部判断框架

Agent 内部使用固定评判框架，但外部不强制展示全部分析。

内部维度包括：

- 行动：用户做了什么
- 情绪：用户经历了什么
- 意义：为什么值得记住
- 特质：体现了什么品质
- 记忆重量：这段经历有多重要

用户可以展开查看 Agent 的判断依据。

### 5.3 奖章外层信息

奖章外层只展示高度概括的信息：

- 奖章图
- 奖章名称
- 一句话授奖理由
- 记忆重量

奖章外层用于成就感、收藏和社交展示。

### 5.4 记忆重量

奖章不使用“普通 / 稀有 / 史诗”这种传统游戏等级，而是使用“记忆重量”表达经历的重要程度。

记忆重量由 Agent 初步判断，用户可以修改。

示例：

- 轻量记忆：日常小确幸、小完成
- 中量记忆：有明确行动、挑战或情绪波动
- 重量记忆：对个人有转折、突破或长期影响

### 5.5 奖章详情

点击奖章后进入详情页。

详情页展示：

- 经历完整叙事
- 关键情节
- Agent 提炼的意义
- Agent 判断依据
- 原始对话 / 原始素材入口
- 隐私设置
- 编辑和重生成入口

详情页里的具体情节由 Agent 生成。默认根据经历自动选择叙事风格，用户可以切换为：

- 日记式
- 故事式
- 档案式

Agent 先生成经历故事草稿，再通过对话和用户共同打磨。

## 6. 奖章编辑与重生成

### 6.1 用户可编辑内容

奖章生成后，用户可以修改：

- 视觉样式
- 公开范围
- 标题
- 授奖理由
- 详情叙事
- 隐藏字段

如果内容被用户编辑过，系统记录“用户编辑过”，用于透明性和版本管理。

### 6.2 意义重生成

当用户觉得 Agent 没有理解准确时，可以进行意义重生成。

用户可以：

- 选择快捷方向，如更偏行动、情绪、特质、故事感
- 用自然语言告诉 Agent 哪里没抓准

系统重新生成奖章，并保留历史版本。

### 6.3 视觉重生成

当用户觉得奖章“不好看”时，可以只重生成视觉。

视觉重生成只改变外观，不改变奖章主题、含义和核心授奖理由。

用户可以：

- 选择风格标签
- 用自然语言描述视觉方向

## 7. 视觉生成

奖章视觉由 Agent 根据用户经历的内容、情绪、氛围和意义自动生成。

产品上不强调固定模板，而强调“这枚奖章是根据你的经历生成的”。

底层可以保留设计约束，以保证统一审美和展示质量。

## 8. 个人主页与长期档案

个人主页包含：

- 奖章墙
- 时间线
- 个人成就档案

默认先展示奖章墙，强化展示感。

### 8.1 时间线

MVP 先以“每次经历”为时间线单位。

每个重要经历都可以生成一个主奖章。

日总结、周总结、月总结后续扩展。

### 8.2 个人成就档案

长期沉淀维度包括：

- 经历类型
- 人格特质
- 情绪轨迹
- 成长关键词

第一版先实现基础提取和展示，后续增强分析深度。

## 9. 连续记录与阶段性产出

产品需要连续记录机制，但弱化打卡压力。

重点不是“必须每天完成”，而是“经历正在形成轨迹”。

阶段性产出包括：

- 周 / 月总结
- 阶段性大奖章
- 个人成长画像更新

MVP 先做阶段性大奖章和简短总结。

阶段性大奖章支持：

- 按周 / 月自动生成
- 用户主动触发
- Agent 发现明显阶段主题时主动建议

Agent 主动程度由用户设置。

默认克制，只在 App 内提示；用户允许后再使用站内提醒或推送通知。

## 10. 社交系统

产品有社交流，用户可以看到别人公开展示的奖章。

每个奖章发布时，用户选择展示范围。

默认公开的是奖章外层内容：

- 奖章图
- 奖章名称
- 短理由
- 记忆重量

用户可以额外公开：

- 经历摘要
- 详情内容
- 部分原始素材

### 10.1 社交流结构

社交流首页采用多 Tab：

- 关注
- 最新
- 热门
- 相似
- 为你推荐

MVP 可以先把入口做出来，推荐质量后续逐步增强。

### 10.2 互动方式

长期支持：

- 点赞 / 收藏
- 评论
- 轻互动
- 关注

MVP 先做轻互动，用更贴合经历成就的表达替代普通点赞。

示例：

- 为你鼓掌
- 我也经历过
- 这很勇敢
- 这值得记住

### 10.3 社交关系

同时支持关注和好友。

- 关注用于订阅公开内容。
- 好友用于更私密的可见范围，比如“仅好友”。

共鸣关系不进入第一版，作为后续迭代。

## 11. 隐私与公开范围

产品默认偏公开展示，但需要保护用户敏感经历。

默认公开的是奖章外层内容。

详情叙事、原始素材、人物地点等敏感信息默认需要用户确认后才公开。

公开范围分为：

- 公开
- 仅好友
- 仅自己
- 自定义隐藏部分内容

第一版支持单独隐藏：

- 人名
- 地点
- 原始图片 / 音频 / 视频
- 原始对话
- 详情叙事
- 情绪标签

时间、学校 / 公司等后续迭代。

## 12. 身份系统

第一版采用匿名 / 昵称制，不要求真实身份。

用户可以设置：

- 昵称
- 头像
- 个人简介

## 13. Agent 设置

Agent 支持预设 + 自由编辑。

用户可以设置：

- 性格
- 对话风格
- 身份设定
- 虚拟形象
- 主动提醒程度

Agent 虚拟形象第一版先做头像 / 立绘。

语音对话、Agent 语音回复、可动虚拟形象放到后续迭代。

## 14. 后续迭代方向

- 上传式经历输入
- 视频上传
- 完整语音通话
- Agent 语音回复
- 可动虚拟形象
- 更强的相似经历推荐
- 共鸣关系
- 周报 / 月报 / 年度回顾
- 更复杂的个人成长画像
- 更丰富的奖章视觉生成能力
- 更精细的隐私字段控制

## 15. 当前待继续确认的问题

下一步建议继续确认：

1. MVP 里每个页面的具体信息架构。
2. 创建经历的对话流程细节。
3. 奖章生成结果的数据结构。
4. 社交流内容卡片长什么样。
5. 第一版技术栈和项目结构。
